All of the Tensorflowjs examples use modules, which works great for build tools and reducing the number of global variables and can be made with only a few changes to work with pure Javascript, but has CORS issues (Cross-Origin Resource Sharing ) when run from your local machine. (Which is strange since all the files are on the local machine but it is a security issue that is a pain to change on Chrome) My solution to this issue is to use namespacing in the external js files. For namespacing to work you have to change any regular functions into inline or arrow functions. My solution works really well as the code used to make the web page is the exact code that any user sees when they "Right click-View page source" on the page. With build tools the final code is very different from the original Javascript or Typescript
